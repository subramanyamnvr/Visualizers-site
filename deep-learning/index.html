Positional Encoding

Transformers process all tokens simultaneously with no built-in notion of sequence order — without positional encodings, "the cat sat" and "sat cat the" would produce identical representations. The original Transformer injects position information by adding a fixed sinusoidal signal to each token embedding: PE(pos, 2i) = sin(pos/10000^(2i/d)) and PE(pos, 2i+1) = cos(pos/10000^(2i/d)). Each dimension of the embedding oscillates at a different frequency — low dimensions change rapidly (capturing fine-grained local order), high dimensions change slowly (capturing global position). This design means relative distances are encoded as predictable phase shifts, allowing the model to generalize to sequence lengths not seen during training. Learned positional embeddings (used in GPT-2) work similarly but treat position as a trainable lookup table, which performs comparably but doesn't extrapolate as cleanly.

Multi-Head Attention

Attention asks: given a query vector, which keys are most relevant, and what are their associated values? Single-head attention computes scores as softmax(QKᵀ/√d_k)V — the scaling by √d_k prevents the dot products from growing too large in high dimensions, which would push softmax into near-zero gradient regions. Multi-head attention runs h independent attention operations in parallel, each with its own learned projections W_Q, W_K, W_V. This lets different heads specialize: one head might attend to the syntactic head of a phrase, another to coreference antecedents, another to the previous token. The h outputs are concatenated and projected back to d_model via W_O. With h=8 heads and d_model=512, each head operates in a 64-dimensional subspace — far cheaper than one 512-dimensional head, and empirically much more expressive.

LayerNorm + Residual Connection

Deep networks suffer from internal covariate shift — the distribution of each layer's input changes as earlier layers update, forcing later layers to constantly re-adapt. Layer Normalization fixes this by normalizing each token's d-dimensional feature vector to zero mean and unit variance, then applying learned scale γ and shift β parameters: LN(x) = γ·(x−μ)/σ + β. Unlike Batch Norm, it operates over the feature dimension (not the batch), making it independent of batch size and sequence length — ideal for variable-length sequences and small-batch fine-tuning. Residual (skip) connections add the block's input directly to its output: x' = x + F(x). This means gradients can flow straight through the addition gate without vanishing, enabling training of 100+ layer networks. Modern transformers use Pre-LN (normalize before the sublayer) rather than Post-LN (the original paper's order), which stabilizes training and removes the need for learning rate warmup in many cases.

KV Cache

Autoregressive generation decodes one token at a time: to generate token t, the model must compute attention over all t−1 previous tokens. Without caching, this requires re-computing the Key and Value matrices for every previous token at every new step — cost grows as O(T²). The KV Cache stores K and V tensors for all previously processed tokens in GPU memory. At each new step, only the new token's Q, K, V are computed; the cached K and V are retrieved and concatenated. This reduces the per-step cost from O(T²) to O(T), enabling fast generation. The tradeoff is memory: for a 7B model with 32 layers, 32 heads, d=128, and a sequence of 2048 tokens in fp16, the KV cache alone is ~2GB. This is why KV cache management (PagedAttention, quantized KV cache, GQA) has become a major area of LLM inference research.

Causal (Masked) Attention

Causal (masked) attention enforces the autoregressive property: when predicting token t, the model should only see tokens 0…t−1, never the future. This is implemented by adding −∞ to attention logits at all positions (i, j) where j > i before the softmax — those positions become exactly 0 after softmax, effectively blocking information flow from future tokens. This masking is applied once as a fixed matrix and requires no additional computation. Without it, training would allow the model to 'cheat' by reading the answer directly from future tokens, making the task trivially easy and the model useless for generation. Encoder models (BERT) use full attention (no mask) since they process the entire sequence at once for representation rather than generation. The three paradigms — causal (GPT), bidirectional (BERT), and prefix-LM (T5) — differ only in which attention mask is applied.

Weight Tying

In a language model, the input embedding matrix W_E ∈ R^(V×d) maps token IDs to vectors, and the output 'unembedding' matrix W_U ∈ R^(d×V) maps the final hidden state back to logits over the vocabulary. These two matrices serve symmetric roles — W_E projects discrete → continuous, W_U projects continuous → discrete over the same vocabulary. Press & Wolf (2017) showed that tying W_E = W_Uᵀ (sharing the same weights) reduces parameters by ~20-30% for large vocabularies, improves perplexity, and acts as a regularizer. Intuitively, a token's input representation and output representation should be similar: if 'Paris' frequently follows 'capital of France', its embedding as an input and its role as an output target should encode the same semantic content. This trick is used in GPT-2, GPT-Neo, and most modern autoregressive LLMs.

B · Core Architectures
Convolutional Neural Network

Convolutional Neural Networks exploit the spatial structure of images through two key inductive biases: translation equivariance (detecting a cat in the top-left vs bottom-right uses the same filter) and local connectivity (nearby pixels are more related than distant ones). A convolutional layer slides a learnable filter W of shape (k×k×C_in) across the input, computing a dot product at each position to produce one output pixel — this produces C_out feature maps for C_out filters. Stacking convolutional layers builds a hierarchy: early layers detect edges and colors, middle layers detect textures and parts, late layers detect whole objects. Pooling (max or average) downsamples spatial resolution, increasing the receptive field and providing some translation invariance. Batch normalization between layers and ReLU activations are standard. AlexNet (2012) proved deep CNNs dominate hand-crafted features; ResNets (2015) enabled 100+ layer networks via skip connections.

LSTM / GRU Gates

Vanilla RNNs update hidden state as h_t = tanh(W_h·h_{t-1} + W_x·x_t), but gradients flow through this recurrence at every step — for long sequences, repeated multiplication by W_h causes them to vanish (if |W_h|<1) or explode (if |W_h|>1). LSTMs introduce a cell state c_t that runs like a 'highway' through time with only multiplicative interactions: f_t = σ(W_f·[h_{t-1},x_t]+b_f) controls what to forget from c_{t-1}; i_t controls what new information to write; o_t controls what to read out. Gradients can flow through the cell state with minimal decay over hundreds of steps. GRUs simplify the LSTM to two gates (reset and update) and merge cell + hidden state, achieving comparable performance with fewer parameters. Both architectures were largely replaced by transformers for NLP after 2018, but remain valuable for streaming, low-latency, and hardware-constrained applications.

Variational Autoencoder (VAE)

A standard autoencoder maps input x to a latent code z = Encoder(x) and reconstructs x̂ = Decoder(z). The problem: the latent space has no structure — nearby points in z-space may decode to completely different outputs, making generation by sampling impossible. VAEs solve this by encoding to a distribution q(z|x) = N(μ, σ²) instead of a point. The decoder then samples z ~ q(z|x) and reconstructs. The ELBO (Evidence Lower BOund) loss has two terms: reconstruction loss (make x̂ ≈ x) and KL divergence (make q(z|x) ≈ N(0,I), regularizing the latent space to be smooth and continuous). The reparameterization trick makes this differentiable: instead of sampling z directly, write z = μ + σ·ε where ε ~ N(0,I) — the randomness is moved outside the gradient computation. At inference, sample ε ~ N(0,I), decode, and get a new image. VAEs are the backbone of latent diffusion models (Stable Diffusion).

Generative Adversarial Network (GAN)

GANs train two networks in a zero-sum game: the Generator G maps noise z ~ p(z) to fake samples G(z), while the Discriminator D tries to classify real samples as 1 and fake samples as 0. G is trained to fool D (maximize log D(G(z))). Theoretically, at equilibrium, G produces samples from the true data distribution and D outputs 0.5 everywhere. In practice, training is notoriously unstable: mode collapse (G generates only a few modes), vanishing gradients (D becomes too good), and oscillation are common. Key improvements: Wasserstein GAN (WGAN) replaces the JS divergence with the more stable Wasserstein distance; Progressive GAN grows resolution gradually; StyleGAN separates content and style via AdaIN. GANs produce extremely sharp, realistic samples but lack explicit likelihoods — you can sample but not evaluate the density of a given point. They've been largely superseded by diffusion models for image synthesis but remain important for tasks needing fast single-pass generation.

Diffusion Model

Diffusion models learn to reverse a gradual noising process. In the forward process, Gaussian noise is added over T steps until the original image becomes pure noise: q(x_t|x_{t-1}) = N(x_t; √(1-β_t)·x_{t-1}, β_t·I). Crucially, the forward process has a closed form: x_t = √ᾱ_t·x_0 + √(1-ᾱ_t)·ε, letting you jump directly to any noise level. The model (a U-Net) is trained to predict the noise ε added at each step given (x_t, t). At inference, start from x_T ~ N(0,I) and iteratively denoise. Why do they work so well? The denoising objective is equivalent to score matching — learning the gradient of the log-data-density ∇_x log p(x), which guides samples toward high-probability regions. DDIM (2020) discovered deterministic sampling: 50 steps instead of 1000 with equal quality. Latent Diffusion (Stable Diffusion) runs the process in a compressed VAE latent space, reducing compute by 8-16×.

C · Training Mechanics
Backpropagation + Chain Rule

Backpropagation is the chain rule applied recursively across a computational graph. Given a scalar loss L, we want ∂L/∂W for every parameter W. The key insight: by the chain rule, if L = f(g(W)), then ∂L/∂W = (∂L/∂f)·(∂f/∂g)·(∂g/∂W) — the upstream gradient times the local Jacobian. Backprop does a single backward pass, reusing intermediate computations to compute all parameter gradients in O(parameters) time — the same cost as one forward pass. Without it, computing gradients by finite differences would cost O(parameters) forward passes. During the forward pass, each node saves its inputs (needed to compute local Jacobians in the backward pass). The backward pass starts at L with gradient 1.0 and propagates through the graph, multiplying by local Jacobians at each node. Modern deep learning frameworks (PyTorch autograd, JAX) build dynamic computational graphs that make this automatic and efficient.

C · Training Mechanics
Optimizers: SGD → Adam → AdamW

Gradient descent updates parameters as θ ← θ − η·∇L, but the raw gradient is noisy (stochastic) and poorly scaled across dimensions. Adam (Adaptive Moment Estimation) maintains two running statistics for each parameter: m_t = β₁m_{t-1} + (1−β₁)g_t (first moment, momentum) and v_t = β₂v_{t-1} + (1−β₂)g_t² (second moment, uncentered variance). The update is θ ← θ − η·m̂_t/√(v̂_t+ε), where m̂ and v̂ are bias-corrected. Parameters with consistently large gradients get small updates; parameters with small or noisy gradients adapt quickly. AdamW separates weight decay from the adaptive update: weight decay in Adam incorrectly interacts with the adaptive learning rates, giving less regularization to parameters with large gradients. AdamW fixes this by applying weight decay directly to the parameter: θ ← θ − η·(m̂_t/√(v̂_t+ε) + λθ). AdamW is the default for training transformers; SGD with momentum still often wins for vision tasks with careful tuning.

C · Training Mechanics
Loss Functions

The loss function defines the learning objective — what the model is trying to minimize. MSE (mean squared error) penalizes large errors quadratically and is ideal for regression where outliers should be heavily penalized. MAE (mean absolute error) is more robust to outliers but has a non-smooth gradient at zero. Cross-entropy loss H(y, p) = −Σ y_i log(p_i) is the standard for classification — it's equivalent to minimizing KL divergence between the true label distribution and predicted probabilities, and connects directly to maximum likelihood estimation. Focal Loss (Lin et al., 2017) addresses class imbalance in object detection: FL = −(1−p_t)^γ log(p_t), where the (1−p_t)^γ factor down-weights easy examples (large p_t) so the model focuses on hard negatives. Huber loss is a smooth combination of MSE (for small errors) and MAE (for large errors), giving robustness without the discontinuous gradient.

Learning Rate Schedulers

The learning rate is arguably the most important hyperparameter in deep learning. Too large: training diverges. Too small: training stagnates. LR schedulers change the learning rate over the course of training. Linear warmup starts with a small LR (avoiding large gradient updates at initialization when parameters are random) and linearly increases to the peak LR over a few thousand steps. Cosine annealing then smoothly decays the LR following a half-cosine curve: η_t = η_min + 0.5·(η_max−η_min)·(1+cos(πt/T_max)). The Transformer paper introduced a specific schedule: η ∝ min(step^(-0.5), step·warmup_steps^(-1.5)) — called the 'Noam scheduler'. OneCycleLR (fast.ai) uses a single cycle of warmup then cosine decay and finds the optimal LR with a learning rate finder. Cyclic LR can escape local minima by oscillating. Step and exponential decay are simpler but less adaptive. For LLM fine-tuning, cosine with warmup (1-5% of training) is the near-universal default.

C · Training Mechanics
Batch Norm vs Layer Norm

Batch Normalization normalizes across the batch dimension: for each feature, compute the mean and variance over the current mini-batch, normalize, then apply learnable scale γ and shift β. This dramatically stabilizes training because each layer sees inputs with consistent statistics regardless of what earlier layers are doing. It also acts as a regularizer (the batch statistics add noise, similar to dropout) and allows much higher learning rates. However, BatchNorm requires large batch sizes to estimate statistics accurately, fails on small batches, and behaves differently at train vs test time (using running statistics at test). Layer Normalization solves this by normalizing over the feature dimension instead of the batch dimension: each token's d-dimensional vector is normalized independently. This makes LayerNorm independent of batch size and sequence length, which is why it replaced BatchNorm in transformers. Instance Norm normalizes each sample×channel independently (used in style transfer); Group Norm is a middle ground.

C · Training Mechanics
Dropout + Regularization

Dropout is a regularization technique that randomly sets a fraction p of neurons to zero during each forward pass of training. The key effect: each training step sees a different 'thinned' sub-network, forcing the network to learn redundant representations — no single neuron can rely on specific other neurons always being present. This prevents co-adaptation and complex coadaptation of feature detectors. Srivastava et al. (2014) showed dropout is equivalent to training an exponentially large ensemble (2^n models for n neurons) and averaging their predictions at test time. At test time, dropout is disabled and activations are scaled by (1−p) to compensate for the expected number of active units. Inverted dropout scales by 1/(1−p) during training instead, keeping test-time inference unchanged. In practice, dropout is most effective in fully-connected layers (p=0.5) and less effective in convolutional layers. For transformers, it's typically applied after attention and FFN sublayers with p=0.1.

D · Foundations
Activation Functions

Without non-linear activation functions, stacking linear layers produces just another linear transformation — no matter how deep the network, it can only represent linear functions. Activations introduce non-linearity, enabling the universal approximation theorem. ReLU (max(0,x)) was transformative: it doesn't saturate for positive inputs (unlike sigmoid/tanh which squash large values to ~1, killing gradients), is computationally trivial, and produces sparse activations. Its main failure mode is 'dead ReLU': neurons that output 0 for all inputs stop receiving gradient and never recover. GELU (Gaussian Error Linear Unit) = x·Φ(x) smoothly gates activations based on Gaussian probability — it's differentiable everywhere and performs better than ReLU in transformers (used in GPT-2, BERT, and most LLMs). SiLU/Swish = x·σ(x) is similar but unbounded below, allowing a small amount of negative signal to pass. SwiGLU combines SiLU with a gated linear unit and is the activation of choice in LLaMA, PaLM, and most frontier models.

MLP & Universal Approximation

A Multi-Layer Perceptron is the fundamental building block of deep learning: alternating linear transformations (matrix multiplications + biases) and non-linear activation functions. Each layer computes h = σ(Wx + b). A network with a single hidden layer and enough neurons can approximate any continuous function on a compact domain (Universal Approximation Theorem, Cybenko 1989, Hornik 1991). However, width alone is insufficient in practice — depth provides exponential expressivity gains, allowing a deep network with d neurons per layer to represent functions that would require exponentially many neurons in a shallow network. Every transformer feedforward block (FFN) is a 2-layer MLP: x → Linear(4d) → GELU → Linear(d). The first layer projects to a higher-dimensional 'expansion space' where the model can represent richer feature combinations, and the second projects back. Researchers have hypothesized that MLP layers act as 'memory' (storing factual associations) while attention layers act as 'routing' (deciding which memories to retrieve).

Vanishing Gradients

In backpropagation, gradients flow from the output layer back through each layer by the chain rule: ∂L/∂W₁ = ∂L/∂h_n · (∂h_n/∂h_{n-1}) · … · (∂h₂/∂h₁) · ∂h₁/∂W₁. For sigmoid activations, the derivative σ'(x) = σ(x)(1−σ(x)) has a maximum of 0.25. In a 100-layer network, the gradient at the first layer is multiplied by ~0.25^100 ≈ 10^{-60} — effectively zero. ReLU partially solves this: its derivative is exactly 1 for positive inputs. Residual connections solve it more thoroughly: x' = x + F(x) creates a direct gradient highway — the gradient of x' with respect to x is always at least 1, regardless of what F computes. BatchNorm/LayerNorm prevents gradient explosion by keeping activations in a well-conditioned range. The exploding gradient problem (gradients growing exponentially) is addressed by gradient clipping: if ‖g‖ > threshold, scale g down to g = threshold · g/‖g‖. This is standard in transformer training (clip_grad_norm = 1.0).

Weight Initialization

Weight initialization seems like a detail but it determines whether a network trains at all. Zero initialization is catastrophic: all neurons compute identical gradients and remain identical forever (symmetry breaking never occurs). Random initialization from N(0, σ²) with a poorly-chosen σ causes exponential signal growth or decay across layers. The key insight (Xavier/Glorot, 2010): to keep the variance of activations constant across layers, initialize weights from N(0, 2/(n_in+n_out)), where n_in and n_out are the fan-in and fan-out of the layer. This ensures that the forward-pass signal neither explodes nor vanishes. He initialization (2015) adapts this for ReLU: since ReLU zeros out ~50% of units, variance is halved — compensate by using N(0, 2/n_in). For transformers with residual connections, the residual stream acts as a sum over many layers, so weight initialization is scaled further (by 1/√(2·num_layers)) to keep the residual stream variance stable. Orthogonal initialization (initialize weight matrices as random orthogonal matrices) is particularly effective for deep RNNs.

Softmax & Temperature

Softmax converts raw logits z into a probability distribution: p_i = exp(z_i) / Σ_j exp(z_j). It's smooth and differentiable (unlike argmax), with all outputs summing to 1. The temperature parameter T scales the logits before softmax: p_i = exp(z_i/T) / Σ_j exp(z_j/T). At T=1: the original distribution. As T→0: the distribution becomes a delta function on the argmax (deterministic). As T→∞: the distribution approaches uniform (maximum randomness). Temperature is crucial for controlling generation quality and diversity: too low (T<0.7) produces repetitive, safe outputs; too high (T>1.2) produces incoherent text. Top-p sampling + temperature is the standard for LLM text generation. In knowledge distillation, a high temperature (T=3-5) on the teacher's softmax reveals 'dark knowledge' — the relative probabilities of wrong answers encode semantic similarity between classes.

E · Attention Variants
Self vs Cross-Attention

Attention is defined by three inputs: Query Q, Key K, and Value V — and the output is softmax(QKᵀ/√d_k)·V. Self-attention means all three come from the same sequence: Q = K = V = the input sequence (after projection). This allows every token to directly gather information from every other token in the same sequence — enabling the long-range dependencies that make transformers so powerful for language. Cross-attention uses queries from one sequence and keys/values from a different sequence. In encoder-decoder transformers (T5, Whisper, BART), the decoder's cross-attention layers attend to the encoder's output: Q comes from the decoder's current hidden states, K and V come from the encoder's hidden states. This is the mechanism by which the decoder 'reads' the source sequence at each generation step. Structurally the operations are identical — only the origin of Q, K, V differs. Cross-attention is also central to diffusion models: the U-Net attends to text embeddings via cross-attention to condition generation on a text prompt.

RoPE (Rotary Position Embedding)

Sinusoidal positional encoding adds absolute position vectors to embeddings, which makes generalization to longer sequences difficult — the model learns absolute positions, not relative ones. Rotary Position Embedding (RoPE) encodes relative positions directly into the attention mechanism. The key idea: rotate the Q and K vectors by an angle proportional to their absolute position before computing the dot product. Because Q_m · K_n = (R_m·q)·(R_n·k) = q·(R_{n-m}·k), the dot product naturally depends only on the relative distance n−m, not absolute positions m or n. The rotation matrix R is block-diagonal with 2×2 rotation matrices at different frequencies, similar to sinusoidal encoding. This means RoPE gets relative position for free from the attention mechanism itself, without adding anything to the embeddings. RoPE enables much better length extrapolation — LLaMA, Mistral, Gemma, Qwen, and virtually all modern open-source LLMs use RoPE. Further extensions (YaRN, LongRoPE) allow extending the context window beyond the training length.

Flash Attention

Standard attention requires materializing the N×N attention score matrix in GPU HBM (main GPU memory): for N=8192 tokens, this is 8192²×2 bytes = 128GB in fp16 — impossible on a single GPU. I/O between HBM and SRAM (the fast on-chip cache) is the actual bottleneck, not FLOPs. FlashAttention (Dao et al., 2022) computes attention in tiles that fit in SRAM, fusing the softmax, dropout, and matmul operations so the full N×N matrix is never materialized. The trick is tiled softmax: process blocks of Q and K, maintaining a running softmax denominator that's corrected block by block (online softmax algorithm). This reduces HBM memory from O(N²) to O(N) while producing exactly the same output as standard attention — no approximation. FlashAttention-2 improved parallelism across sequence length. FlashAttention-3 targets Hopper GPUs. All frontier models (GPT-4, Llama, Claude, Gemini) use FlashAttention as their attention kernel.

E · Attention Variants
GQA / MQA

In standard Multi-Head Attention (MHA), each of the h attention heads has its own Q, K, V projection matrices — requiring h separate K and V tensors to be stored in the KV cache. For a 70B model with 64 heads and long contexts, this KV cache becomes enormous. Multi-Query Attention (MQA, Shazeer 2019) takes this to the extreme: all Q heads share a single K head and a single V head. This reduces the KV cache by h×, dramatically cutting memory and memory bandwidth. Grouped Query Attention (GQA, Ainslie et al. 2023) is the pragmatic middle ground: group Q heads into G groups, each group sharing one K and one V. With G=1, GQA = MQA; with G=h, GQA = MHA. Llama 2 70B uses GQA with G=8 (64 Q heads, 8 K/V heads), reducing KV cache by 8× with minimal quality degradation. Virtually all modern production LLMs (Mistral, Llama 3, Gemma, Qwen) use GQA.

F · Transformer Variants
Encoder / Decoder / Seq2Seq

The choice of transformer architecture determines what task a model is suited for. Encoder-only models (BERT, RoBERTa) apply fully bidirectional attention — every token attends to every other token. Trained with Masked Language Modeling (predict randomly masked tokens), they learn rich contextual representations ideal for classification, NER, question answering, and semantic similarity. They cannot generate text. Decoder-only models (GPT, LLaMA, Mistral) apply causal attention — each token only attends to past tokens. Trained with next-token prediction (causal language modeling), they're generative. Modern LLMs are almost universally decoder-only because CLM scales better than MLM, and in-context learning emerges in large decoder models. Encoder-decoder models (T5, BART, Whisper) combine a bidirectional encoder and a causal decoder connected via cross-attention. Ideal for sequence-to-sequence tasks where input and output are both sequences (translation, summarization, ASR). The encoder reads the full input; the decoder generates the output token by token, attending back to encoder states via cross-attention.

F · Transformer Variants
FFN & SwiGLU

Every transformer layer has two sublayers: multi-head attention and a feedforward network (FFN). The FFN is a position-wise 2-layer MLP applied independently to each token: FFN(x) = ReLU(xW₁ + b₁)W₂ + b₂. With expansion factor 4, the inner layer is 4×d — in GPT-3 with d=12288, the FFN has ~600M parameters per layer, making the FFN blocks account for ~2/3 of all transformer parameters. Noam Shazeer (2020) showed that replacing ReLU with GLU variants — Gated Linear Units, where part of the activation gates the other part — substantially improves perplexity. SwiGLU = Swish(xW) ⊗ (xV), applied as FFN(x) = SwiGLU(x,W,V)·W₂. This requires three weight matrices instead of two but is now standard in LLaMA, PaLM, Mistral, Gemma, and most frontier models. The FFN layers are now widely understood as 'key-value memories': the first layer projects to a 4d space where each dimension can be seen as a 'memory key', and the second layer retrieves associated 'values'.

Scaling Laws

Scaling laws are empirical power-law relationships that describe how model performance improves with scale. Kaplan et al. (2020) showed that for language models, cross-entropy loss follows L(N) = (N_c/N)^(α_N) and L(D) = (D_c/D)^(α_D), where N is parameters, D is training tokens, and the exponents α ≈ 0.076. Crucially, the loss curves are smooth and predictable — meaning you can train a small model to forecast what a much larger model will achieve. The Chinchilla paper (Hoffmann et al., 2022) overturned the prevailing practice of training large models on fixed token budgets: for a fixed compute budget C ∝ N·D, the optimal allocation is equal compute spent on parameters and data, giving the Chinchilla optimal: D_optimal ≈ 20·N (train a model on 20 tokens per parameter). Llama models followed this — Llama-1 7B is trained on 1T tokens (far more than Chinchilla optimal), making it more inference-efficient. Scaling laws have become the central tool for planning training runs at frontier labs.

G · Fine-Tuning & PEFT
LoRA (Low-Rank Adaptation)

Full fine-tuning of a 7B parameter model requires ~56GB GPU memory (weights + gradients + optimizer states in fp32) — impractical for most researchers. LoRA (Low-Rank Adaptation) freezes all pre-trained weights and injects trainable low-rank decompositions into the weight matrices: for a weight W₀ ∈ R^(d×k), add ΔW = B·A where B ∈ R^(d×r) and A ∈ R^(r×k) with r << min(d,k). The forward pass becomes h = W₀x + (BA)x/r·α, where α is a scaling factor. This reduces trainable parameters from d×k to r×(d+k) — for r=16, d=4096, k=4096, that's 16×8192 = 131K vs 16.7M parameters: a 128× reduction. The key insight: pre-trained large language models have a low intrinsic dimensionality — the task-specific update ΔW has low rank in practice. After training, BA can be merged into W₀ for zero inference overhead. LoRA is applied typically to Q and V projection matrices in attention (or all 4: Q, K, V, O), and sometimes to FFN layers.

QLoRA

QLoRA (Dettmers et al., 2023) combines 4-bit quantization of the frozen base model with LoRA adapters to enable fine-tuning of 65B+ parameter models on a single 48GB GPU. Three innovations make this work: (1) NF4 (NormalFloat 4-bit): a new data type optimal for normally-distributed weights — it uses 4-bit quantization levels spaced so that each bin contains equal probability mass under a normal distribution, minimizing quantization error for Gaussian weight distributions. (2) Double quantization: quantize the quantization constants themselves, saving ~0.37 bits/parameter. (3) Paged optimizers: use NVIDIA unified memory to page optimizer states to CPU RAM during memory spikes, preventing OOM errors. The LoRA adapters are kept in BF16. Gradients during backprop flow through the quantized weights via the straight-through estimator (treat quantization as identity during backward pass). QLoRA made fine-tuning frontier-scale models accessible to academic researchers and enabled the explosion of open-source fine-tuned LLMs in 2023.

SFT & Instruction Tuning

A base language model trained with next-token prediction on internet text learns to complete text in whatever style is most common, but doesn't naturally follow instructions — it might continue a question rather than answering it, or continue a story when asked for a summary. Supervised Fine-Tuning (SFT) adapts a base model to instruction-following by training on (instruction, response) pairs. Crucially, the loss is applied only to the response tokens — not the instruction tokens — using a loss mask. This teaches the model to generate responses conditioned on instructions rather than continue them. The data format (prompt template) matters enormously: how the instruction and response are formatted, what special tokens are used (e.g., [INST]/[/INST] in Llama, <|im_start|>/<|im_end|> in ChatML), and whether multi-turn conversations are included all affect the resulting model behavior. Instruction tuning (Wei et al., 2022) showed that fine-tuning on a diverse mixture of instructional tasks across domains dramatically improves zero-shot generalization to new tasks.

G · Fine-Tuning & PEFT
Prompt & Prefix Tuning

Instead of updating any model weights, prompt tuning prepends a small set of trainable 'soft tokens' (continuous vectors in embedding space, not actual words) to the input sequence and trains only these tokens. The frozen model learns to condition on these vectors, which can encode task-specific information. With enough model scale (T5-11B), prompt tuning matches full fine-tuning performance with just 20K trainable parameters vs 11B. Prefix Tuning (Li & Liang, 2021) extends this to every layer: trainable prefix vectors are prepended to K and V in every attention layer, giving the model more degrees of freedom to condition on the task. Adapters take a different approach: small bottleneck modules (Linear down → nonlinearity → Linear up) are inserted inside each transformer block. Only the adapter parameters are trained (~0.5-5% of total), while the rest is frozen. P-Tuning v2 found that prefix tuning applied to all layers performs as well as fine-tuning for many NLP tasks, and all these methods are now unified under the PEFT (Parameter-Efficient Fine-Tuning) umbrella.

H · Alignment & RLHF
RLHF Pipeline

Instruction tuning makes a model helpful but doesn't teach it human preferences: what makes a response not just correct but also safe, concise, honest, and well-formatted? RLHF (Christiano et al., 2017; applied at scale by OpenAI) has three stages. First, supervised fine-tuning: fine-tune the base model on high-quality (instruction, response) pairs. Second, reward modeling: generate multiple responses to prompts, have human annotators rank them by preference, and train a reward model (the SFT model with a scalar head) to predict these preference rankings via the Bradley-Terry model loss. Third, RL fine-tuning: use the reward model as a reward signal to fine-tune the policy with PPO. A KL penalty β·KL(π_θ ‖ π_SFT) is added to prevent the model from diverging too far from the SFT model (which would produce reward hacking — finding adversarial inputs that fool the reward model). This full pipeline is what transformed GPT-3 (a base model) into InstructGPT/ChatGPT.

H · Alignment & RLHF
DPO (Direct Preference Optimization)

PPO-based RLHF is notoriously complex: it requires maintaining 4 models simultaneously (reference policy, live policy, reward model, value model), involves sensitive hyperparameter tuning, and is prone to reward hacking. DPO (Direct Preference Optimization, Rafailov et al., 2023) eliminates the reward model entirely. The key insight: there exists a closed-form relationship between the optimal policy and the implicit reward function. By substituting this into the RLHF objective, the optimization becomes a supervised learning problem directly on preference pairs (prompt, chosen response, rejected response). The DPO loss is: L = -log σ(β·log[π_θ(y_w|x)/π_ref(y_w|x)] - β·log[π_θ(y_l|x)/π_ref(y_l|x)]), where y_w is the preferred ('winner') response, y_l is the rejected ('loser') response, π_ref is the frozen reference policy (SFT model), and β controls the KL penalty strength. DPO directly increases the log-probability of preferred responses and decreases the log-probability of rejected responses, weighted by how 'surprising' this is relative to the reference model. It's simpler, more stable, and computationally cheaper than PPO, and has become the dominant alignment method.

H · Alignment & RLHF
Reward Modeling

The reward model is the operationalization of human preferences as a learnable function. Given a prompt x and a response y, the reward model outputs a scalar r(x,y) representing how much a human would prefer this response. Training data: annotators are shown the same prompt with 2-4 different responses (from the SFT model or other sources) and rank them. These rankings are converted to pairwise comparisons and used to train the reward model with a Bradley-Terry loss: L = -log σ(r(x, y_w) - r(x, y_l)) for each (preferred, rejected) pair. The reward model is typically initialized from the SFT model and has a linear head added that projects the final hidden state to a scalar. A key challenge: reward hacking — the PPO policy can find 'adversarial' inputs that fool the reward model into giving high scores without actually being good responses. KL penalty, reward model ensembles, and iterative human feedback (Constitutional AI) are techniques to combat this.

Quantization

A standard FP32 weight takes 4 bytes; an FP16 weight takes 2 bytes; INT8 takes 1 byte; INT4 takes 0.5 bytes. Quantization maps continuous-valued weights to a discrete grid: q(x) = round(x / scale) + zero_point. The scale and zero_point are calibration constants that map the observed weight range to the quantization grid. Post-Training Quantization (PTQ) quantizes a model after training, requiring a small calibration dataset to compute scale/zero_point per layer. GPTQ (Frantar et al., 2023) uses second-order information (inverse Hessian) to minimize the quantization error layer by layer — it can quantize to 3-4 bits with minimal perplexity loss. AWQ (Lin et al., 2023) protects the 1% of 'salient' weights (those with the highest activation magnitudes, identified by a small calibration set) by keeping them at higher precision, dramatically improving accuracy. NF4 (used in QLoRA) is optimized for normally-distributed weights. A 7B model in FP16 requires ~14GB; in 4-bit it's ~4GB — fitting on a single consumer GPU.

Knowledge Distillation

Knowledge distillation (Hinton et al., 2015) trains a small 'student' model to mimic a large 'teacher' model. The key insight: the teacher's soft probability distribution over classes contains far more information than the hard one-hot label. If an image is 70% cat, 20% lion, 5% dog, the soft labels reveal semantic relationships between classes — information that's lost in the hard label 'cat'. The student minimizes a combination of: (1) cross-entropy with hard labels (standard supervision), and (2) KL divergence between student and teacher soft outputs at temperature T (soft supervision). Temperature T>1 softens the teacher's distribution, preventing a few high-probability classes from dominating and making the dark knowledge (small but informative probabilities) more accessible. For LLMs, sequence-level distillation (seq-KD) generates 'silver' training data from the teacher and trains the student on it with standard language modeling loss. Distillation is how GPT-4 capabilities are transferred to GPT-4o-mini, and how Phi-3 achieves surprisingly strong performance at small scale.

Speculative Decoding

Autoregressive generation is memory-bandwidth-bound: the large model generates one token per forward pass, and each forward pass must load all model weights from HBM. Speculative decoding (Leviathan et al., 2023; Chen et al., 2023) exploits the fact that a much smaller draft model can predict what the large 'verifier' model would generate, most of the time. The algorithm: (1) the draft model auto-regressively generates k tokens (fast); (2) the verifier model processes all k tokens in a single parallel forward pass (batching makes this almost as fast as a single token); (3) tokens are accepted greedily where the draft and verifier agree (above some probability threshold), and rejected otherwise — with a correction term to maintain the exact verifier distribution; (4) generation continues from the first rejected position. When the acceptance rate is high, this generates k+ tokens for the cost of ~2 forward passes, achieving 2-3× speedup with mathematically identical output distribution. Smaller models from the same family (e.g., Llama-3.1-8B as draft for Llama-3.1-70B) have the highest acceptance rates.

J · Modern Architectures
ResNet & Skip Connections

Before ResNets (He et al., 2016), adding more layers beyond ~20 led to higher training error — not overfitting, but actual optimization failure. The degradation problem: with random initialization, a 56-layer network is harder to optimize than a 34-layer one, even though the deeper network should at least match the shallower one by learning identity mappings in extra layers. The fix is elegant: instead of learning H(x), let each block learn the residual F(x) = H(x) − x. The block output becomes F(x) + x (the skip connection adds the input directly). Now the optimizer only needs to learn small corrections from identity — if no transformation is needed, just push F(x) → 0. This makes optimization dramatically easier and allowed training of 152-layer networks that beat all previous methods on ImageNet. Skip connections also solve the vanishing gradient problem: gradients can flow directly through skip connections without passing through the block, maintaining signal strength all the way to the first layer. Every modern deep architecture — transformers (residual around attention and FFN), DenseNets, U-Nets — uses skip connections.

J · Modern Architectures
Mixture of Experts (MoE)

In a standard transformer, every token is processed by every parameter at every layer. Mixture of Experts replaces the dense FFN block with N expert FFN networks and a learned router that selects top-k experts per token. For each input token x, the router computes g = softmax(x·W_gate), selects the top-k expert indices, and computes output = Σᵢ∈top-k gᵢ · Expert_i(x). This decouples total parameter count from computation per token: a 8×7B MoE model (Mixtral) has 47B total parameters but only activates ~13B per token — giving the quality of a 47B model at the inference cost of a 13B model. Key challenge: load balancing. If the router consistently sends all tokens to the same few experts, others get no gradient and remain untrained. A load balancing auxiliary loss encourages uniform expert utilization. Sparse MoE is used in GPT-4, Gemini 1.5, Mixtral, DeepSeek, and most frontier-scale models. Switch Transformer (Fedus et al., 2021) showed top-1 routing (k=1) scales to 1.6 trillion parameters.

J · Modern Architectures
Mamba / SSM

Transformers scale quadratically with sequence length due to the N×N attention matrix. State Space Models (SSMs) represent sequence transformations as linear dynamical systems: h'(t) = Ah(t) + Bx(t), y(t) = Ch(t) + Dx(t), where h is a continuous hidden state. Discretized with step size Δ, they become efficient recurrences. The problem: classic SSMs (S4, HIPPO) have fixed A and B matrices — they treat all tokens equally regardless of content. Mamba (Gu & Dao, 2023) introduces selective state spaces (S6): A, B, C, Δ are all functions of the input x_t. This selectivity allows the model to decide what information to propagate through the state and what to discard — similar to LSTM gates but with more principled theoretical backing. The parallelism trick: selective SSMs can be computed efficiently in parallel during training using the parallel scan algorithm (analogous to prefix sums), then run as a cheap O(1) recurrence at inference. Mamba achieves transformer-quality performance on language tasks with O(n) inference cost and O(1) per-step memory.

K · Embeddings
Tokenization (BPE)

Neural networks operate on fixed-size numerical inputs, but raw text is a variable-length string of characters. Tokenization converts text to a sequence of integer IDs. Character-level tokenization (one token = one character) has a small vocabulary (~256) but sequences are very long and the model must learn to compose characters into words. Word-level tokenization (one token = one word) has a large vocabulary (~100K+) with shorter sequences but can't handle out-of-vocabulary words or morphological variations. BPE (Byte Pair Encoding, Sennrich et al., 2016) starts with individual characters and iteratively merges the most frequent pair of adjacent tokens into a new token, stopping at a target vocabulary size. This gives a vocabulary of subwords: common words like 'the' become one token; rare words like 'unhappiness' are split as 'un', 'happiness'; novel words are decomposed to known subwords. GPT-4 uses a BPE tokenizer with 100K vocabulary. SentencePiece (used in LLaMA/T5) works directly on raw text (no pre-tokenization by whitespace), enabling language-agnostic tokenization. Tokenization efficiency (tokens/character) directly affects context window capacity.

Word2Vec

Word2Vec (Mikolov et al., 2013) showed that simple neural networks trained on word co-occurrence produce vector representations with remarkable algebraic properties: king − man + woman ≈ queen; Paris − France + Italy ≈ Rome. Two architectures: CBOW (Continuous Bag of Words) predicts a center word from its surrounding context words, averaging their embeddings as input; Skip-gram predicts surrounding context words given the center word. Both are shallow networks (one hidden layer), and the learned hidden layer weights become the word embeddings. Training uses negative sampling — for each positive (word, context) pair, sample k negative (word, random_word) pairs and train a binary classifier, avoiding the expensive full softmax over the entire vocabulary. The embeddings encode the distributional hypothesis: words that appear in similar contexts have similar meanings. GloVe (Pennington et al., 2014) achieves similar results by explicitly factorizing the word-word co-occurrence matrix. Both were largely replaced by contextual embeddings (BERT) but remain relevant as baselines and for understanding the geometry of semantic spaces.

Contrastive Learning

Contrastive learning trains representations by pushing similar (positive) pairs together and dissimilar (negative) pairs apart in embedding space, without requiring manual labels. SimCLR (Chen et al., 2020): take an image, apply two random augmentations (crop, color jitter, blur) to create two 'views'. Their embeddings should be similar to each other and dissimilar to all other images in the batch. The NT-Xent loss: L = -log[exp(sim(zᵢ,zⱼ)/τ) / Σₖ≠ᵢ exp(sim(zᵢ,zₖ)/τ)]. The batch acts as a source of negatives — larger batches provide more negatives and stronger training signal (SimCLR uses batch size 4096-8192). CLIP (Radford et al., 2021) applies this idea to image-text pairs: the image encoder and text encoder are trained to align corresponding image-text pairs while pushing apart non-corresponding pairs. The result is a shared embedding space where images and text descriptions are directly comparable — enabling zero-shot image classification, text-guided image retrieval, and serving as the backbone of all modern text-to-image models.

RAG (Retrieval-Augmented Generation)

LLMs have a knowledge cutoff (they don't know about events after training) and hallucinate facts from their parametric memory. RAG (Lewis et al., 2020) addresses this by augmenting generation with retrieved relevant documents. At inference: (1) the query is encoded into a dense vector using a retriever model (often a bi-encoder like SBERT); (2) approximate nearest-neighbor search finds the k most similar document chunks from a vector database (FAISS, Pinecone, Weaviate); (3) retrieved chunks are prepended to the prompt as context; (4) the LLM generates conditioned on both the query and retrieved context. The retriever can be a sparse system (BM25), a dense bi-encoder, a re-ranker (cross-encoder for second-stage ranking), or a combination (Hybrid RAG). Advanced RAG variants include HyDE (generate a hypothetical document, retrieve similar ones), multi-hop RAG (iterative retrieval for complex questions), and FLARE (generate → check confidence → retrieve when uncertain). RAG trades off latency (extra retrieval step) for factuality and knowledge freshness.

Decoding Strategies

The output logits of a language model define a probability distribution over the next token. How you sample from this distribution drastically affects generation quality. Greedy decoding always picks the highest-probability token — deterministic but prone to repetition and local optima. Beam search maintains k candidate sequences ('beams'), expanding each with all possible next tokens and keeping the top k by cumulative probability. It produces higher likelihood sequences but they're often stilted and generic (Holtzman et al. showed high-likelihood ≠ good text). Top-k sampling (Fan et al., 2018): restrict sampling to the top k most likely tokens, renormalize, and sample. This avoids very unlikely tokens but k is a fixed threshold regardless of the actual distribution shape. Top-p (nucleus) sampling (Holtzman et al., 2020): instead, keep the smallest set of tokens whose cumulative probability ≥ p. This dynamically adjusts the number of candidates — if the model is confident (one token has p=0.99), only that token is sampled; if uncertain, many tokens are included. Top-p sampling with p=0.9 and temperature T=0.8 is the most common production default for open-ended generation.

Vision Transformer (ViT)

CNNs have a useful inductive bias (translation invariance, local connectivity) but it also limits them — they struggle to capture long-range spatial dependencies. The Vision Transformer (Dosovitskiy et al., 2020) applies the standard transformer to images with minimal modification. An image is divided into fixed-size patches (typically 16×16 pixels), each patch is flattened and linearly projected to a d-dimensional embedding. A learnable [CLS] token is prepended, and 1D positional embeddings are added to all tokens. The resulting sequence is fed to a standard transformer encoder. For ImageNet classification, the [CLS] token's output is passed through an MLP head. ViT shows that with enough data (JFT-300M) and scale, transformers match or exceed CNNs, without inductive bias. At smaller data scales, CNNs still win — ViT overfits without sufficient data or heavy regularization. DeiT (2021) showed that with knowledge distillation and augmentation, ViT matches CNNs on ImageNet with only ImageNet training data. ViT became the backbone for CLIP, DALL-E, MAE, DINO, SAM, and virtually all modern vision models.

U-Net

U-Net (Ronneberger et al., 2015) was designed for biomedical image segmentation, where you need to classify each pixel and can only afford a few hundred training images. Its key innovation: an encoder-decoder architecture with skip connections that bypass the bottleneck at every resolution level. The encoder (contracting path) downsamples the image with conv+pooling, progressively extracting higher-level features while losing spatial detail. The decoder (expanding path) upsamples with transposed convolutions, recovering spatial resolution. At each resolution level, the encoder feature map is directly concatenated with the decoder feature map (skip connection), allowing the decoder to use both high-level semantic context (from the deep bottleneck) and low-level spatial details (from the shallow encoder). This resolves the tension between 'what' and 'where': the bottleneck captures global semantics, the skip connections preserve fine-grained location. U-Net became the de facto denoiser in DDPM and Stable Diffusion — the U-Net is the neural network inside the diffusion model that predicts the noise, conditioned on the timestep and text embeddings via cross-attention.

CLIP

Before CLIP, vision models were trained on fixed, hand-labeled datasets (ImageNet's 1000 classes). CLIP (Radford et al., 2021) showed that instead of predicting a fixed set of labels, you can supervise a vision model with natural language. Trained on 400M (image, text) pairs scraped from the internet, CLIP jointly trains an image encoder (ViT) and a text encoder (Transformer) with a contrastive objective: for a batch of N pairs, the image and text encoders produce N embeddings each, and the training pushes the N correct (image, text) pairs together while pushing the N²−N incorrect pairs apart. At inference, zero-shot classification: encode candidate labels as text ('a photo of a cat'), encode the query image, find the text embedding with the highest cosine similarity. No additional training needed for new classes or tasks. Object Detection (YOLO vs R-CNN)

Object detection localizes and classifies multiple objects in an image, outputting bounding boxes (x,y,w,h) and class labels for each detected object. DETR (2020) eliminated anchor boxes and NMS entirely using a transformer with learned object queries.

M · Computer Vision
Swin Transformer

Plain ViT processes all patches globally — every patch attends to every other patch at every layer. For dense prediction tasks (detection, segmentation) this is problematic: the O(N²) attention cost for high-resolution images is prohibitive, and there's no hierarchical feature representation (all layers produce the same resolution). Swin Transformer (Liu et al., 2021) addresses both: it computes self-attention within local windows of patches (7×7 windows), making attention cost O(N) with respect to image size. Shifted window partitioning: alternate layers shift the window grid by half the window size, creating cross-window connections on even layers. This allows information to flow between windows without global attention. Hierarchical feature maps: 4 stages with patch merging that progressively downsamples resolution and increases channels (like ResNet stages), creating multi-scale representations ideal for detection and segmentation. Swin achieved state-of-the-art on ImageNet, COCO detection, and ADE20K segmentation, and is the backbone in many production vision systems.

N · Reinforcement Learning
MDP & Q-Learning / DQN

Reinforcement Learning formalizes sequential decision-making as a Markov Decision Process (MDP): at each timestep t, an agent observes state s_t, takes action a_t, receives reward r_t, and transitions to s_{t+1}. The agent's goal is to maximize expected cumulative discounted reward G_t = Σ_k γ^k r_{t+k}, where γ ∈ [0,1) discounts future rewards. Q-Learning learns the action-value function Q(s,a) = E[G_t | s_t=s, a_t=a] via the Bellman equation: Q(s,a) ← Q(s,a) + α(r + γ max_a' Q(s',a') − Q(s,a)). At convergence, the optimal policy is π*(s) = argmax_a Q*(s,a). DQN (Mnih et al., 2015) replaced the Q-table with a deep neural network, enabling RL from raw pixel inputs. Two stabilizing tricks: a replay buffer (store past transitions and sample random minibatches to break temporal correlation) and a target network (a periodically-updated frozen copy of Q used to compute Bellman targets, preventing moving target instability). DQN achieved superhuman performance on 49 Atari games, launching modern deep RL.

N · Reinforcement Learning
Policy Gradient & PPO

Q-learning learns the value of actions; policy gradient methods directly optimize the policy π_θ(a|s) by gradient ascent on expected return. The REINFORCE algorithm (Williams, 1992): sample a trajectory τ = (s_0, a_0, r_0, …), compute returns G_t at each step, then update θ ← θ + α Σ_t G_t ∇_θ log π_θ(a_t|s_t). The gradient log π_θ(a_t|s_t) is the 'direction to make action a_t more likely'; it's weighted by G_t, so actions in high-return trajectories are reinforced. PPO (Proximal Policy Optimization, Schulman et al., 2017) addresses REINFORCE's high variance and instability. The key insight: we want to improve the policy but not change it too drastically (large updates can catastrophically harm performance). PPO clips the probability ratio r_t(θ) = π_θ(a_t|s_t)/π_θ_old(a_t|s_t) to stay within [1-ε, 1+ε] (ε=0.2 typically). The clipped objective: L = E[min(r_t A_t, clip(r_t, 1-ε, 1+ε) A_t)], where A_t is the advantage. PPO is simple, stable, and sample-efficient — it's the RL algorithm used in RLHF for training ChatGPT and Llama 2.

N · Reinforcement Learning
Actor-Critic & GAE

REINFORCE has high variance: the return G_t is a noisy single-trajectory estimate of the expected return, requiring many samples to estimate gradient direction accurately. Actor-Critic methods reduce this variance by using a learned value function (the critic V(s)) as a baseline. The advantage A(s,a) = Q(s,a) − V(s) measures how much better action a is than average — it has lower variance than raw returns because the critic provides a relatively stable baseline. The actor updates to increase probability of high-advantage actions; the critic updates to better predict returns. A2C runs multiple workers in parallel (synchronously), collecting diverse experience to stabilize training. A3C uses asynchronous updates. GAE (Generalized Advantage Estimation, Schulman et al., 2016) provides a tunable bias-variance tradeoff for advantage estimation: Â_t^GAE(γ,λ) = Σ_l (γλ)^l δ_{t+l}, where δ_t = r_t + γV(s_{t+1}) − V(s_t) is the TD error. λ=1 gives full Monte Carlo returns (low bias, high variance); λ=0 gives TD(0) (high bias, low variance). λ=0.95 is the standard in PPO implementations.

O · Generative Models (Ext)
Normalizing Flows

Normalizing flows are a class of generative models that learn an exact, tractable bijective mapping between a simple distribution (Gaussian) and a complex data distribution. The change-of-variables formula gives the exact log-likelihood: log p(x) = log p_z(f(x)) + log|det(J_f(x))|, where J_f is the Jacobian of the transformation f. The key requirement: f must be invertible and its Jacobian determinant must be efficiently computable. Coupling layers (RealNVP, Dinh et al., 2016) achieve this: split input x into [x₁, x₂]; transform x₂ as x₂' = x₂ · exp(s(x₁)) + t(x₁) while keeping x₁ unchanged. This produces a triangular Jacobian with determinant = Σ s(x₁) — computed in O(d) time. Stacking many coupling layers builds complex transformations. Unlike GANs (no likelihood), VAEs (approximate ELBO), and diffusion models (iterative), normalizing flows provide exact likelihood and exact sampling in a single pass. They're widely used in density estimation, anomaly detection, and as priors in VAEs. Flow Matching (2023) generalizes this to ODE-based flows along straight paths, used in Stable Diffusion 3.

O · Generative Models (Ext)
Classifier-Free Guidance (CFG)

Diffusion models conditioned only on text prompts (via cross-attention) often produce blurry, uncommitted outputs because the model's training objective doesn't push it to maximize fidelity to the condition — just to model the data distribution. Classifier-Free Guidance (Ho & Salimans, 2022) dramatically sharpens conditioning by mixing conditional and unconditional scores. During training, the condition (text prompt) is randomly dropped with probability p_uncond ≈ 0.1, training both a conditional p(x|c) and unconditional p(x) model in one network. At inference, the guided denoising direction is: ε_guided(x_t, c) = ε_θ(x_t, ∅) + w · (ε_θ(x_t, c) − ε_θ(x_t, ∅)), where ε_θ(x_t, c) is the conditional noise prediction, ε_θ(x_t, ∅) is the unconditional, and w is the guidance scale. This extrapolates in the direction of the text condition: at w=1, no guidance; at w=7.5 (typical), high text fidelity; at w=15+, oversharpened. The same technique applies to any condition: class label, style, depth map. CFG is used in virtually every production image and video generation model (DALL-E 2, Stable Diffusion, Imagen, Sora).

The original diffusion models used U-Net as the denoiser — an architecture with inductive biases from CNNs (local convolutions, hierarchical features) that work well but don't scale as cleanly as transformers. DiT is the architecture used in Stable Diffusion 3, Flux, and OpenAI's Sora-class video models. Its clean scaling behavior with compute makes it the clear path forward for generative models.

Masked Autoencoders (MAE)

BERT's masked language modeling masked ~15% of tokens and achieved outstanding language representations. He et al. (2021) showed the same idea works for vision but requires masking a much higher fraction — 75% — because image patches are far more redundant than text tokens (adjacent patches are highly correlated, making low-masking reconstruction trivially solvable by interpolation). MAE's design is asymmetric: the encoder processes only the visible 25% of patches (making it 4× cheaper per image during pretraining), while a lightweight decoder reconstructs all patches including the masked ones. Mask tokens (learnable vectors representing 'missing patch') are fed only to the decoder. The reconstruction target is raw pixel values (or normalized pixel values), not deep features — surprisingly, this simple target works as well or better than complex perceptual losses. The learned encoder representations transfer excellently to ImageNet classification, detection, and segmentation. The key insight: image understanding requires holistic reasoning, not just local texture matching — by masking 75% of an image and reconstructing it, the model must understand scene structure.

P · Self-Supervised Learning
DINO / DINOv2

DINO (Self-DIstillation with NO labels, Caron et al., 2021) is a self-supervised framework where a student network learns to match the output of a teacher network. The key design choices that make it work: (1) the teacher is not trained by gradient — it's updated via exponential moving average of the student weights: θ_teacher ← m·θ_teacher + (1−m)·θ_student, where m≈0.996. This prevents collapse (both networks collapsing to constant outputs) because the teacher is always slightly 'ahead'. (2) The student processes local crops of the image; the teacher processes global crops. The student must predict the teacher's output for the global view from only a local view — forcing it to understand the global semantic context from local evidence. (3) A centering operation on teacher outputs further prevents collapse. The resulting attention maps of DINO's [CLS] token have a remarkable emergent property: without any supervision, they segment objects from backgrounds — the self-supervised model discovers semantic parts by attending to them. DINOv2 scaled this up with curated data and produces the best general-purpose visual features, used as backbones in depth estimation, robotics, and multimodal models.

P · Self-Supervised Learning
BYOL (Bootstrap Your Own Latent)

SimCLR and MoCo require large batches of negative pairs to work — the negatives provide the repulsive force that prevents the encoder from mapping everything to the same point (representational collapse). BYOL (Bootstrap Your Own Latent, Grill et al., 2020) showed that you don't need any negatives at all. The key: two asymmetric branches. The online network has an extra 'predictor' MLP on top of the projector. The target network is the EMA of the online network (no gradient). Loss: minimize 2 − 2 cos(online_pred, target_proj). Why doesn't this collapse? The asymmetry between online and target (predictor only in online, EMA update only in target) prevents the trivial solution. Interestingly, if you remove either the predictor or the EMA and use gradient propagation for both, collapse occurs immediately. BYOL achieves excellent ImageNet representations with batch size 256 (vs SimCLR's 4096), making it far more practical. DINO, MoCo v3, and most modern SSL frameworks are variations on BYOL's student-teacher + EMA insight.

R · Training Infrastructure
Gradient Accumulation & Mixed Precision

Large batch training is empirically beneficial (more stable gradients, better generalization) but GPU memory limits practical batch size. Gradient accumulation simulates a larger batch by performing multiple forward-backward passes before a single optimizer step: accumulate gradients over N micro-batches, then call optimizer.step() and zero_grad(). The effective batch size = N × micro_batch_size. This comes at no extra memory cost (only one micro-batch's activations in memory at once) but N× compute cost per update. Mixed precision training stores model weights in fp32 (for numerical stability of optimizer) but performs forward and backward passes in fp16 or bf16 (half the memory, 2-8× faster on Tensor Cores). A key challenge: fp16 has a limited dynamic range and can underflow (small gradient values become 0). Loss scaling addresses this: multiply the loss by a large scale factor S before backward, divide gradients by S before optimizer step, keeping gradients in the representable range. BF16 (bfloat16) has the same dynamic range as fp32 (8 exponent bits) but less precision (7 mantissa bits vs 23 for fp32) — it's less prone to overflow/underflow and has become the default for LLM training on modern GPUs (A100, H100).

Distributed Training (DDP / ZeRO)

Training large models requires distributing computation across multiple GPUs, typically via one of three parallelism strategies. Data Parallel (DDP): each GPU has a full model replica and processes a different data shard. After the backward pass, gradients are all-reduced (summed and averaged) across all GPUs using ring-allreduce, so all replicas update identically. Scales linearly with GPUs for compute but not for model size (each GPU must fit the full model). ZeRO (Zero Redundancy Optimizer, Rajbhandari et al., 2020) addresses this by sharding model states across GPUs — eliminating the redundancy in DDP. ZeRO Stage 1: shard optimizer states (~12 bytes/param in Adam). ZeRO Stage 2: + shard gradients. ZeRO Stage 3: + shard parameters (each GPU holds only 1/N of the model). ZeRO Stage 3 enables training models much larger than a single GPU's memory, at the cost of extra communication (parameters must be gathered on demand). FSDP (Fully Sharded Data Parallel) is PyTorch's native implementation of ZeRO Stage 3. Tensor parallelism (Megatron-LM) splits individual weight matrices across GPUs — the matrix multiply is split and recombined. All three strategies combine in '3D parallelism' for 1000+ GPU training runs.

KV Cache & PagedAttention (vLLM)

During LLM inference, the KV cache grows linearly with sequence length and batch size. Serving many concurrent requests means storing their KV caches simultaneously — but naive static allocation pre-reserves the maximum sequence length for each request, wasting most of this memory (internal fragmentation). vLLM (Kwon et al., 2023) introduced PagedAttention, inspired by OS virtual memory. The KV cache is divided into fixed-size blocks (pages), typically 16 tokens per page. A block table maps each request's logical token positions to physical block addresses, which need not be contiguous. New pages are allocated on demand and freed when a request completes. This allows nearly 100% KV cache utilization. Combined with continuous batching (iterative scheduling) — where finished sequences are immediately replaced by new requests rather than waiting for the whole batch to complete — vLLM achieves 24× higher throughput than static batching. PagedAttention also enables copy-on-write sharing of KV cache pages for parallel sampling (multiple outputs from the same prefix share the same KV cache pages).

GradCAM & Saliency Maps

Neural network predictions are often opaque — you know the answer but not why. GradCAM (Gradient-weighted Class Activation Mapping, Selvaraju et al., 2017) produces coarse localization maps that highlight the image regions most important for a CNN's prediction. The method: compute the gradient of the class score y^c with respect to the final convolutional feature maps A^k (a tensor of spatial maps). Global average pool the gradient to get importance weights α_k^c = (1/Z) Σᵢⱼ (∂y^c/∂A^k_{ij}). The class activation map is L^c_GradCAM = ReLU(Σ_k α_k^c · A^k). ReLU is applied because we only want features that have a positive effect on the class score. The resulting map is coarse (feature map resolution) but highlights which spatial regions the model focused on. Guided Backpropagation improves sharpness by passing only positive gradients backward through ReLU layers. Integrated Gradients (Sundararajan et al., 2017) computes the integral of gradients along a straight-line path from a baseline (black image) to the input, providing attribution scores with axiomatic properties (completeness, sensitivity). These tools are critical for debugging models and detecting spurious correlations.

Model Calibration

A model is well-calibrated if its confidence scores match its empirical accuracy: when it predicts 80% confidence, it should be correct ~80% of the time. Modern neural networks are notoriously overconfident — they output high-confidence predictions even when wrong. A reliability diagram plots predicted confidence (x-axis, binned) vs actual accuracy (y-axis); a perfectly calibrated model lies on the diagonal. Expected Calibration Error (ECE) quantifies this: ECE = Σ_m (|B_m|/n)|acc(B_m) − conf(B_m)|, weighted average of bin calibration errors. Temperature scaling (Guo et al., 2017) is the simplest post-hoc calibration: tune a single temperature T on a validation set to soften the logits. Despite its simplicity (single scalar), it consistently improves calibration without degrading accuracy. Platt scaling and isotonic regression are alternative post-hoc methods. Calibration is critical for high-stakes decisions (medical, legal) where the model's confidence should be interpreted as a reliable probability. LLMs are particularly poorly calibrated — their verbalized confidence often doesn't match their actual token probability.

T · Math Foundations
Information Theory

Information theory provides the mathematical foundation for deep learning objectives. Shannon entropy H(p) = −Σ p(x) log p(x) measures the average uncertainty in a distribution — uniform distribution has maximum entropy; a delta function has zero entropy. Entropy has units of bits (base-2 log) or nats (natural log). Cross-entropy H(p,q) = −Σ p(x) log q(x) measures the average number of bits needed to encode events from distribution p using a code optimized for q. The training loss in classification is cross-entropy between true labels p and model predictions q — minimizing it drives q toward p. KL divergence KL(p‖q) = Σ p(x) log(p(x)/q(x)) = H(p,q) − H(p) measures how much extra bits q requires compared to the true p. It's asymmetric (KL(p‖q) ≠ KL(q‖p)) and always ≥ 0. KL appears everywhere: VAE regularization (KL(posterior ‖ prior)), RLHF KL penalty (KL(policy ‖ reference)), DPO objective, information bottleneck. Mutual Information MI(X;Y) = H(X) − H(X|Y) = KL(p(x,y) ‖ p(x)p(y)) measures how much knowing Y reduces uncertainty about X — the core quantity in contrastive learning and representation learning theory.

Bias-Variance Tradeoff & Double Descent

The bias-variance decomposition explains the fundamental tension in statistical learning: E[(y−ŷ)²] = Bias² + Variance + Noise. Bias measures how far the average prediction (over different training sets) is from the true function — high bias means underfitting (model too simple). Variance measures how much the prediction changes between different training sets — high variance means overfitting (model memorizes training data, doesn't generalize). Classically, the optimal model sits in a U-shaped test error curve: too simple → high bias, too complex → high variance, with the best model at the minimum. Modern deep learning breaks this classical picture. The 'double descent' phenomenon (Belkin et al., 2019; Nakkiran et al., 2020): as model complexity increases past the interpolation threshold (where the model can fit training data exactly), test error first peaks (variance spike) then decreases again as the model becomes even more overparameterized. This is the regime modern neural networks operate in: billions of parameters, training error near zero, but excellent generalization. The implicit bias of gradient descent (toward minimum-norm solutions) and the inductive bias of the architecture explain why overparameterized models generalize.

OOD Detection & Distributional Shift

Machine learning models are trained on a specific distribution p_train(x,y) and are typically deployed on inputs from the same distribution. Distribution shift occurs when test inputs come from a different distribution. Covariate shift: p(x) changes but p(y|x) stays the same (e.g., model trained on sunny day images deployed on rainy night images). Label shift: p(y) changes but p(x|y) stays the same. Dataset shift is the general term. Out-of-distribution (OOD) detection asks: given an input, does it look like it's from the training distribution? The goal is not to make predictions on OOD inputs, but to detect and abstain. Baseline method: max softmax probability (MSP) — inputs from the training distribution tend to have higher max softmax probabilities. Problem: neural networks are overconfident even on OOD inputs. Better methods: Mahalanobis distance from class mean embeddings (Lee et al., 2018), energy-based scores, ODIN (temperature scaling + input perturbation). The AUROC on OOD detection benchmarks measures how well a detector separates in-distribution from OOD inputs. Handling distributional shift is one of the key unsolved challenges for safe deployment of ML systems.

In-Context Learning (ICL)

In-context learning (ICL) is one of the most surprising emergent capabilities of large language models: by providing a few examples of a task in the prompt (without any gradient updates), the model can perform that task on new inputs. For example, giving 3 translation pairs (English→French) lets GPT-3 translate new sentences without any fine-tuning. ICL emerged at around 1B-10B parameters — smaller models show near-zero ICL performance, and performance scales sharply with model size (emergence). The mechanism is debated. One view: ICL is implicit Bayesian inference over the space of possible tasks, weighted by how often each task appeared in pretraining. Another view: transformer attention layers implement a form of gradient descent in their forward pass — the attention mechanism can effectively perform in-context optimization. Empirically, the specific labels and even the label correctness in demonstrations matter less than the format and distribution of examples (Min et al., 2022). Chain-of-thought prompting extends ICL by including reasoning steps in demonstrations, dramatically improving multi-step reasoning. ICL is why LLMs are so flexible — the same model can solve classification, translation, coding, and math with just prompt formatting.

Graph Neural Networks (GNN)

Most deep learning assumes input is a grid (images) or sequence (text). Graph Neural Networks extend deep learning to graph-structured data: social networks, molecular graphs, knowledge graphs, 3D point clouds. The fundamental operation is message passing: each node aggregates feature vectors from its neighbors, combines them with its own features, and updates its representation. Formally: h_v^(l+1) = UPDATE(h_v^(l), AGGREGATE({h_u^(l) : u ∈ N(v)})). Different GNN variants differ in how AGGREGATE and UPDATE are implemented. GCN (Kipf & Welling, 2017): AGGREGATE = mean over neighbors, normalized by degree. Graph-level output via global mean pooling. The spectral motivation: this is an approximation to graph convolution in the Fourier domain. GAT (Graph Attention Network, Veličković et al., 2018): compute attention coefficients α_ij = softmax(LeakyReLU([W h_i ‖ W h_j])) for each edge, then aggregate as weighted sum. Different neighbors get different importance. GraphSAGE samples a fixed-size neighborhood and learns an aggregation function — enabling inductive learning on new graphs. GNNs power recommendation systems (Pinterest PinSage, Facebook DLRM), drug discovery (AlphaFold contact prediction, molecular property prediction), and fraud detection.

BERT & Masked Language Modeling

BERT (Bidirectional Encoder Representations from Transformers, Devlin et al., 2018) demonstrated that deeply bidirectional pre-training dramatically improves performance across NLP tasks. Unlike GPT's unidirectional (left-to-right) training, BERT sees the full sentence context in every layer. Pretraining task 1 — Masked Language Modeling (MLM): randomly mask 15% of tokens; 80% are replaced with [MASK], 10% with a random token, 10% left unchanged. The model must predict the original token from the surrounding context. The 10% random/unchanged tokens prevent the model from learning 'if [MASK], predict something' and force representations to be contextually useful even for unmasked positions. Pretraining task 2 — Next Sentence Prediction (NSP): given two sentences A and B, predict whether B follows A in the original document (50% real, 50% random). NSP was later shown to be unhelpful (RoBERTa removes it). BERT-base: 12 layers, 768 hidden, 12 heads, 110M params. BERT-large: 24 layers, 1024 hidden, 16 heads, 340M. Fine-tuned BERT achieved state-of-the-art on GLUE, SQuAD, and 11 other NLP benchmarks in 2018, launching the pre-train/fine-tune paradigm that now dominates all of NLP and computer vision.
