01 / Atlas Map

The full Python landscape in one animated route map.

The language grows in layers. Execution and memory shape objects, objects shape types, types power functions and iteration, and that model expands into modules, classes, files, time handling, and the data ecosystem.

The map is arranged as a build order: foundations first, then types, control flow, iteration, modularity, object patterns, standard library tools, and finally the array-table-plotting layer.

02 / Foundations

Interpreter mechanics, environments, memory, and imports.

Python feels simple on the surface because a lot of important machinery is hidden. This section makes the environment, object references, mutability rules, and import model visible.

03 / Types Studio

Numbers, text, sequences, mappings, slicing, and comprehension patterns.

Most Python fluency comes from seeing how a few core protocols repeat: indexing, hashing, truthiness, unpacking, mutability, and iteration.

04 / Functions

Call signatures, closures, decorators, higher-order tools, and introspection.

Python functions are not just reusable chunks of code. They are objects with metadata, scope behavior, callable signatures, and composition patterns that show up everywhere in modern code.

05 / Iteration

Iterables, iterators, generators, itertools, context managers, and coroutines.

Python's lazy tools are where the language stops being merely pleasant and becomes mechanically elegant. They are also the ideas that unlock better pipelines, smaller memory footprints, and cleaner resource management.

06 / Object Model

Classes, properties, descriptors, dunder methods, enums, and exception design.

Python's object model is flexible because attribute access, callable behavior, comparison, construction, and method binding are all part of the language's data model instead of being bolted on later.

07 / Standard Library

Files, serialisation, time, decimal arithmetic, randomness, and collections.

Python ships with a large toolkit that often solves the first practical version of a problem without any external dependency: text files, CSV, JSON, time parsing, exact decimal math, random sampling, and more.

08 / Ecosystem

HTTP, timezone helpers, arrays, tables, and plotting.

After the core language and standard library, the next major layer is the practical toolkit many Python workflows lean on: HTTP requests, better date parsing, array math, dataframe work, and visualization.

09 / Official Docs

Core documentation routes for deeper study.

These are the official Python documentation entry points that map directly to the ideas in this atlas: the tutorial, language reference, data model, built-ins, standard library modules, and class model.
